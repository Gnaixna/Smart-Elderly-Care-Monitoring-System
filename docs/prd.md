# 需求文档

## 1. 应用概述

### 1.1 应用名称
智慧养老健康监护系统小程序

### 1.2 应用描述
本小程序面向居家养老场景，为家属提供远程监护能力。通过与老人端监测设备（基于STM32的多模态安全健康监测系统）配合，实时查看老人的生理参数（心率、血氧、体温）、跌倒状态、环境烟雾浓度等数据，接收异常告警推送，并支持远程调整预警阈值。所有监测数据通过WiFi上传至阿里OSS对象存储和Supabase数据库，小程序通过API获取数据并展示。

---

## 2. 用户与使用场景

### 2.1 目标用户
- 老年人的家属、子女
- 居家养老护理人员

### 2.2 核心使用场景
- 家属远程查看老人实时健康状态
- 接收跌倒、生理异常、烟雾超标等紧急告警
- 查询历史健康数据趋势
- 远程调整心率、血氧、体温预警阈值

## 3. 页面结构与功能说明

### 3.1 页面结构

```
智慧养老健康监护系统小程序
├── 用户登录页
├── Tab导航（底部3个Tab）
│   ├── 实时监测（首页）
│   ├── 历史记录
│   └── 我的
```

### 3.2 页面功能说明

#### 3.2.1 用户登录页

**页面元素**
- 欢迎标题：「欢迎使用 - 智慧养老健康监护系统」
- 登录方式切换：账号登录 / 验证码登录
- 账号登录表单：
  - 账号输入框
  - 密码输入框
  - 记住密码选项
  - 忘记密码链接
- 验证码登录表单：
  - 手机号输入框
  - 验证码输入框
  - 获取验证码按钮
- 登录按钮
- 立即注册入口

**功能说明**
- 支持账号密码登录和手机验证码登录两种方式
- 用户输入账号密码或手机号验证码后点击登录按钮完成登录
- 登录成功后跳转至实时监测页
- 首次使用用户点击「立即注册」进入注册流程

#### 3.2.2 实时监测页（首页）

**页面元素**
- 设备信息区域：
  - 设备名称：老人监护设备001
  - 在线状态：在线/离线（绿色圆点表示在线）
  - 最后更新时间
- 四个监测卡片：
  - 心率卡片：显示数值（次/分）、正常范围60-100
  - 血氧饱和度卡片：显示数值（%）、正常范围95-100
  - 体温卡片：显示数值（°C）、正常范围36.0-37.3
  - 跌倒状态卡片：显示「正常」或「跌倒检测中」
- 烟雾状态卡片：显示「正常」或「异常」，当前环境安全/危险
- 历史数据快捷入口按钮
- 弹窗提醒预警：异常时自动弹出，显示异常类型、数值、时间，包含确认/忽略按钮

**功能说明**
- 每隔3秒自动刷新设备上传的最新数据
- 设备连续30秒以上未上传数据时，页面顶部显示「设备离线」提示
- 跌倒事件或烟雾异常时，对应卡片红色高亮显示并闪烁
- 检测到异常时自动弹出告警弹窗，用户可点击确认或忽略
- 点击历史数据快捷入口跳转至历史记录页

#### 3.2.3 历史记录页

**页面元素**
- 时间维度切换：日/周/月
- 日期选择器
- 数据类型切换：心率/血氧/体温
- 折线图展示区域（带时间轴）
- 报警事件记录列表：
  - 事件类型：跌倒报警、心率过高、体温异常、烟雾异常等
  - 发生时间
  - 具体数值

**功能说明**
- 用户选择时间维度（日/周/月）和日期范围
- 用户选择数据类型（心率/血氧/体温）
- 折线图展示选定时间范围内的数据变化趋势，支持缩放和拖拽
- 报警事件列表按时间倒序排列
- 点击某条报警记录查看详情，跳转至该时间点的数据快照

#### 3.2.4 我的页面

**页面元素**
- 用户信息展示：
  - 头像
  - 昵称
  - 手机号
- 预警阈值设置：
  - 心率上限/下限输入框
  - 血氧下限输入框
  - 体温上限/下限输入框
  - 保存按钮
- 设备管理入口
- 消息通知设置入口
- 退出登录按钮

**功能说明**
- 显示当前登录用户的头像、昵称、手机号
- 用户输入心率上限/下限、血氧下限、体温上限/下限，点击保存后通过云端下发至设备端
- 点击设备管理进入设备绑定/解绑页面
- 点击消息通知设置进入订阅消息授权页面
- 点击退出登录返回登录页

## 4. 业务规则与逻辑

### 4.1 设备绑定规则
- 首次使用小程序需完成设备绑定
- 用户手动输入设备的ProductKey、DeviceName、DeviceSecret，或扫描设备二维码完成绑定
- 绑定信息保存至本地，下次打开自动加载
- 绑定成功后，小程序首页顶部显示设备名称及在线状态

### 4.2 数据获取规则
- 小程序通过调用阿里云物联网平台API获取设备上传的数据
- 实时监测页每隔3秒轮询一次设备属性查询接口
- 历史记录页通过时序数据存储服务（InfluxDB或表格存储）查询指定时间范围内的数据

### 4.3 异常告警规则
- 设备端检测到跌倒事件、生理参数超阈值、烟雾异常时，通过阿里云平台上报事件
- 小程序服务端调用微信订阅消息接口，向用户推送告警消息
- 消息模板包含：设备名称、告警类型（跌倒、心率过高、体温异常、烟雾超标）、发生时间
- 用户点击消息卡片跳转至小程序首页，定位至异常时刻数据
- 若用户未订阅或推送失败，下次打开小程序时弹出未读告警列表

### 4.4 阈值远程调节规则
- 用户在「我的」页面输入心率上限/下限、血氧下限、体温上限/下限
- 点击保存后，小程序通过阿里云平台下发属性设置指令至设备端
- 设备端接收指令后更新本地阈值参数
- 阈值调整成功后，小程序显示「设置成功」提示

### 4.5 数据存储规则
- 设备端通过ESP8266 WiFi模块将监测数据上传至阿里OSS对象存储
- 历史数据和告警记录保存至Supabase数据库
- 小程序通过API从Supabase查询历史数据和报警事件

## 5. 异常与边界情况

| 场景 | 处理方式 |
|------|----------|
| 设备离线超过30秒 | 页面顶部显示「设备离线」提示，提醒用户检查设备或网络连接 |
| 网络请求失败 | 显示「网络异常，请稍后重试」提示，支持手动刷新 |
| 用户未授权订阅消息 | 首次进入小程序时弹出授权请求，若拒绝则无法接收推送 |
| 阈值输入超出合理范围 | 显示「请输入合理的阈值范围」提示，不允许保存 |
| 设备绑定失败 | 显示「绑定失败，请检查设备信息」提示，支持重新输入 |
| 历史数据查询为空 | 显示「暂无数据」提示 |
| 折线图数据点过多 | 自动采样显示，支持缩放查看详细数据 |

## 6. 验收标准

1. 用户打开小程序，完成微信授权登录，输入账号密码或手机验证码，登录成功后进入实时监测页
2. 用户在实时监测页查看设备在线状态、心率、血氧、体温、跌倒状态、烟雾状态，数据每3秒自动刷新
3. 设备端检测到跌倒事件，小程序自动弹出告警弹窗，显示「跌倒检测」、发生时间，用户点击确认关闭弹窗
4. 用户点击历史数据快捷入口，进入历史记录页，选择「日」维度和「心率」数据类型，查看当天心率变化折线图
5. 用户点击「我的」页面，输入心率上限110、下限50，点击保存，显示「设置成功」提示
